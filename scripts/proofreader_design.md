设计一个智能体系统。这个系统要实现以下功能：接受一个参数（当前语言），阅读位于 styleguide 文件夹下面这个语言的风格指南的英文原文和 docs/guides 文件夹下面对应译文，并进行校对工作。

这个智能体需要：
1. 验证翻译是否全面
2. 验证翻译是否易于理解、通顺、本土化，并进行修正
3. 验证代码等例子是否进行了恰当的本地化
3. 如果有需要取舍的地方，列出对应的问题以便人类介入

这个智能体的功能要求：
1. 使用 Claude API 调用最新的 Sonnet。妥善保存 API Key，使得用户不需要重复输入，但是也不能泄露。
2. 进行恰当的上下文管理，以保证 API 不收到过多无用的上下文，但是也要有充分的上下文
3. 使用命令行界面，实时显示状态，让用户等待时能了解目前智能体的状态。至少每秒更新一次，甚至可以流式输出当前模型的对话，但是要有良好的排版。
4. 在 debug/ 文件夹下用恰当的格式记录智能体的所有决策、操作和 API 输入输出，方便后期查阅和调试
5. 使用 Python，在 venv 隔离环境下运行

在 scripts/ 文件夹下新建文件夹并放置你的项目。你的项目应该模块化以方便维护。

====

这个智能体应该更加智能一些。它应该自动判断如何分割文档，同时要有一些状态管理（比如可以更新术语表或者其他需要保持一致的内容）。请更详细地考虑这里的设计，可以搜索一些智能体设计的最佳实践。

你的项目应该可以独立运行，不要引用文件夹以外的内容。
